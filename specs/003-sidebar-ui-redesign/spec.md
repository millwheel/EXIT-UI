## User Scenarios & Testing *(mandatory)*

### User Story 1 - 새로운 사이드바 기반 네비게이션 경험 (Priority: P1)

로그인한 사용자가 흰색 배경의 좌측 사이드바를 통해 계정관리, 광고관리 등 주요 메뉴에 접근하고, 현재 선택된 메뉴가 primary color(#2E7D5A)로 강조 표시되어 현재 위치를 명확히 인지할 수 있다.

**Why this priority**: 전체 애플리케이션의 네비게이션 구조를 변경하는 핵심 기능으로, 다른 모든 UI 요소가 이 사이드바 레이아웃에 의존한다.

**Independent Test**: 사이드바의 메뉴 항목을 클릭하여 페이지 이동이 정상적으로 작동하고, 선택된 메뉴가 시각적으로 구분되는지 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** 사용자가 로그인한 상태에서, **When** 페이지에 접속하면, **Then** 좌측에 흰색 배경의 사이드바가 표시되고 상단 헤더는 표시되지 않는다.
2. **Given** 사이드바가 표시된 상태에서, **When** 광고관리 메뉴를 클릭하면, **Then** 해당 메뉴 항목이 primary color(#2E7D5A) 배경과 rounded 스타일로 강조되고 광고관리 페이지로 이동한다.
3. **Given** 사이드바가 표시된 상태에서, **When** 메뉴 항목에 마우스를 올리면, **Then** 해당 항목의 배경색이 gray 계열로 변경되어 호버 상태임을 나타낸다.

---

### User Story 2 - 사이드바 상단 로고 표시 (Priority: P1)

사용자가 사이드바 최상단에 위치한 로고(K-168.png)를 통해 브랜드 아이덴티티를 인식하고, 로고 클릭 시 홈/대시보드로 이동할 수 있다.

**Why this priority**: 로고는 브랜드 인식과 홈 네비게이션의 핵심 요소로, 사이드바 레이아웃과 함께 구현되어야 한다.

**Independent Test**: 로고가 사이드바 최상단에 표시되고 클릭 시 대시보드로 이동하는지 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** 로그인한 사용자가 페이지에 접속했을 때, **When** 사이드바를 확인하면, **Then** 최상단에 K-168.png 로고가 표시된다.
2. **Given** 사이드바에 로고가 표시된 상태에서, **When** 로고를 클릭하면, **Then** 대시보드(홈) 페이지로 이동한다.

---

### User Story 3 - 프로필 및 로그아웃 영역 재배치 (Priority: P2)

사용자가 사이드바 하단에 위치한 프로필 영역에서 자신의 username을 확인하고, 프로필 클릭 시 프로필 수정 화면으로 이동하며, 하단의 로그아웃 버튼을 통해 로그아웃할 수 있다.

**Why this priority**: 프로필과 로그아웃 기능은 필수 기능이나, 기존 헤더에서 사이드바로 위치만 변경되므로 기능적으로는 기존 구현을 활용할 수 있다.

**Independent Test**: 프로필 영역에서 username이 표시되고, 클릭 시 프로필 수정 페이지로 이동하며, 로그아웃 버튼 클릭 시 로그아웃되는지 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** 로그인한 사용자가 페이지에 접속했을 때, **When** 사이드바 하단을 확인하면, **Then** 프로필 영역에 현재 로그인한 사용자의 username이 표시된다.
2. **Given** 프로필 영역이 표시된 상태에서, **When** 프로필을 클릭하면, **Then** 프로필 수정 페이지(/profile)로 이동한다.
3. **Given** 프로필 아래에 로그아웃 버튼이 표시된 상태에서, **When** 로그아웃 버튼을 클릭하면, **Then** 세션이 종료되고 로그인 페이지로 리다이렉트된다.

---

### User Story 4 - 헤더 제거 및 레이아웃 조정 (Priority: P2)

기존 상단 헤더가 제거되고 전체 콘텐츠 영역이 사이드바 우측으로 재배치되어 화면 공간을 효율적으로 활용할 수 있다.

**Why this priority**: 헤더 제거는 사이드바 중심 레이아웃으로의 전환에 필수적이며, 콘텐츠 영역 조정도 함께 이루어져야 한다.

**Independent Test**: 헤더가 표시되지 않고, 콘텐츠가 사이드바 우측 전체 영역에 표시되는지 확인할 수 있다.

**Acceptance Scenarios**:

1. **Given** 로그인한 사용자가 페이지에 접속했을 때, **When** 화면 상단을 확인하면, **Then** 기존의 헤더 영역이 표시되지 않는다.
2. **Given** 헤더가 제거된 상태에서, **When** 메인 콘텐츠 영역을 확인하면, **Then** 콘텐츠가 사이드바 우측의 전체 영역을 활용하여 표시된다.

---

### Edge Cases

- 매우 긴 username이 입력된 경우 어떻게 표시되는가? → 텍스트가 사이드바 너비 내에서 말줄임표(ellipsis)로 처리된다.
- 화면 크기가 작은 디바이스에서 사이드바가 어떻게 표시되는가? → 현재 스코프에서는 데스크톱 뷰만 고려하며, 반응형 디자인은 향후 개선 사항으로 둔다.
- 로그아웃 API 호출이 실패한 경우 어떻게 처리되는가? → 기존 에러 처리 로직을 유지하며, 사용자에게 적절한 피드백을 제공한다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 기존 상단 헤더를 제거하고 좌측 사이드바 중심의 레이아웃으로 전환해야 한다.
- **FR-002**: 사이드바는 흰색 배경으로 표시되어야 한다.
- **FR-003**: 사이드바 최상단에 로고(K-168.png)가 표시되어야 하며, 클릭 시 대시보드 페이지로 이동해야 한다.
- **FR-004**: 사이드바 메뉴 항목은 다음 순서로 위에서 아래로 배치되어야 한다: 홈 로고, 계정관리, 광고관리, 프로필(username 표시), 로그아웃
- **FR-005**: 선택된(활성화된) 메뉴 항목은 primary color(#2E7D5A) 배경과 rounded 스타일로 표시되어야 한다.
- **FR-006**: 메뉴 항목에 마우스를 올렸을 때(hover) 배경색이 gray 계열로 변경되어 사용자에게 상호작용 가능함을 나타내야 한다.
- **FR-007**: 모든 메뉴 항목의 width와 height는 동일해야 한다.
- **FR-008**: 프로필 영역은 사이드바 하단에 위치하며 현재 로그인한 사용자의 username을 표시해야 한다.
- **FR-009**: 프로필 영역 클릭 시 프로필 수정 페이지(/profile)로 이동해야 한다.
- **FR-010**: 로그아웃 버튼은 프로필 영역 아래에 위치하며, 클릭 시 세션을 종료하고 로그인 페이지로 리다이렉트해야 한다.
- **FR-011**: primary color는 기존의 #2E7D5A 색상을 유지해야 한다.

### Key Entities

- **Sidebar**: 좌측에 고정 배치되는 네비게이션 컴포넌트. 로고, 메뉴 항목, 프로필, 로그아웃 버튼을 포함한다.
- **Menu Item**: 사이드바 내 개별 네비게이션 항목. 아이콘, 레이블, 선택 상태, 호버 상태를 가진다.
- **User Profile**: 사이드바 하단에 표시되는 사용자 정보 영역. username과 프로필 수정 네비게이션을 담당한다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 사용자가 로그인 후 3초 이내에 새로운 사이드바 레이아웃이 완전히 렌더링된다.
- **SC-002**: 사용자가 원하는 메뉴에 1번의 클릭으로 접근할 수 있다.
- **SC-003**: 현재 선택된 메뉴가 시각적으로 명확히 구분되어 사용자가 현재 위치를 즉시 파악할 수 있다.
- **SC-004**: 프로필 수정 및 로그아웃 기능에 기존과 동일하게 2번 이내의 클릭으로 접근할 수 있다.
- **SC-005**: reference 이미지와 동일한 디자인 스타일로 구현되어 시각적 일관성을 달성한다.

## Assumptions

- 데스크톱 뷰(브라우저 기반)만 현재 스코프에 포함되며, 모바일/태블릿 반응형 디자인은 별도 기능으로 다룬다.
- 기존 인증 및 세션 관리 로직은 변경 없이 유지한다.
- primary color(#2E7D5A) 및 관련 색상 팔레트는 기존 디자인 시스템을 따른다.
- 로고 이미지(K-168.png)는 프로젝트 public 폴더에 이미 존재한다.
- 사이드바의 고정 너비는 reference 이미지를 참고하여 적절히 설정한다(대략 240-260px 범위).
